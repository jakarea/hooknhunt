  Chart of Accounts Best Practices                                                                                                
                                                                                                                                  
  üí° Best Practices:                                                                                                              
                                                                                                                                  
  1. Use Consistent Naming                                                                                                        
    - ‚úÖ "Rent Expense"                                                                                                           
    - ‚ùå "Rent" or "Office Rent"                                                                                                  
    - Stick to same naming pattern                                                                                                
  2. Use Account Codes                                                                                                            
    - Makes finding accounts easier                                                                                               
    - Common pattern:                                                                                                             
        - 1xxx = Assets                                                                                                           
      - 2xxx = Liabilities                                                                                                        
      - 3xxx = Equity                                                                                                             
      - 4xxx = Revenue                                                                                                            
      - 5xxx = Expenses                                                                                                           
  3. Don't Create Too Many                                                                                                        
    - Too many accounts = confusion                                                                                               
    - Group similar expenses                                                                                                      
    - Example: One "Utility Expense" not separate "Electricity", "Water", "Gas"                                                   
  4. Don't Create Too Few                                                                                                         
    - Too few = not enough detail                                                                                                 
    - Can't track spending properly                                                                                               
    - Example: Separate "Rent" and "Utilities" not just one "Expenses"                                                            
  5. Keep Accounts Active                                                                                                         
    - Mark inactive if not used                                                                                                   
    - Don't delete - history needed                                                                                               
    - Can reactivate anytime                                                                                                      
                                                                                                                                  
  ‚ö†Ô∏è Warnings:                                                                                                                    
                                                                                                                                  
  1. Don't Delete Accounts                                                                                                        
    - Breaks historical data                                                                                                      
    - Creates errors in reports                                                                                                   
    - Mark inactive instead                                                                                                       
  2. Don't Change Account Types                                                                                                   
    - Asset to Expense = major problem                                                                                            
    - Breaks all reports                                                                                                          
    - Create new account instead                                                                                                  
  3. Don't Mix Types                                                                                                              
    - Keep Asset accounts separate                                                                                                
    - Keep Revenue accounts separate                                                                                              
    - Each account has one type only

  Understanding Exchange Rates                                                                                                    
                                                                                                                                  
  Exchange Rate = How much of one currency equals another                                                                         
                                                                                                                                  
  In this system:                                                                                                                 
  - Default Currency: Bangladeshi Taka (BDT) - ‡ß≥                                                                                  
  - Exchange Rate: How many BDT = 1 unit of foreign currency                                                                      
                                                                                                                                  
  Example:                                                                                                                        
  - USD exchange rate: 110.50                                                                                                     
  - Means: 1 USD = 110.50 BDT                                                                                                     
  - Means: $100 = 11,050 BDT                                                                                                      
                                                                                                                                  
  How it works:                                                                                                                   
                                                                                                                                  
  If you sell product for $500:                                                                                                   
                                                                                                                                  
  Step 1: Record sale in USD: $500                                                                                                
  Step 2: System converts to BDT: $500 √ó 110.50 = 55,250 BDT                                                                      
  Step 3: BDT amount recorded in accounts                                                                                         
                                                                                                                                  
  Why? Because all accounting is done in default currency (BDT)

  Currency Best Practices                                                                                                         
                                                                                                                                  
  üí° Best Practices:                                                                                                              
                                                                                                                                  
  1. Keep Default Currency Stable
