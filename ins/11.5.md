**Goal:** Implement the "Receive Stock" functionality.
**Logic:** A Purchase Order contains **Parent Products**. When receiving, the user must split the received quantity into specific **Variants** (e.g., 100 T-Shirts -> 50 Red, 50 Blue) and add them to Inventory.

---

### PART 1: BACKEND (`hooknhunt-api`)

**Task 1: Helper Endpoint**
Update `ProductController`. Add `getVariants($id)` to fetch all variants for a specific Parent Product ID.
* Route: `GET /api/v1/admin/products/{id}/variants`

**Task 2: Core Logic in `PurchaseOrderController`**
Add a method `receiveStock(Request $request, $id)`.

* **Input:**
    ```json
    {
      "po_item_id": 1, // The specific line item from purchase_order_items
      "items": [
        { "variant_id": 101, "quantity": 50, "unit_cost": 550 }, // Red
        { "variant_id": 102, "quantity": 50, "unit_cost": 550 }  // Blue
      ]
    }
    ```
* **Validation:**
    * Check if `po_item_id` belongs to the order `$id`.
    * Calculate `total_receiving_qty` from the input items.
    * Check: `(current_stocked_qty + total_receiving_qty)` <= `ordered_quantity`. (Prevent over-stocking).
* **Execution (Transaction):**
    1.  Loop through `items`:
        * **Inventory:** Increment `quantity` in `inventory` table for `product_variant_id`. (Create row if not exists).
        * **Price:** Update `landed_cost` in `product_variants` table (optional, based on user input).
    2.  **PO Item:** Increment `stocked_quantity` in `purchase_order_items` table.
    3.  **PO Status:** If ALL items in the PO are fully stocked, set order status to `completed`.

**Task 3: Routes**
* Add `POST /purchase-orders/{id}/receive-stock`
* Add `GET /products/{id}/variants`
